# Style Guidelines

这份文档定义案例和 gold response 的写作标准。

## 核心原则

一个场景应该因为情绪和关系逻辑是真实的而显得有生命力——不是因为用了好看的短语。

## 目标文体

默认的 pilot 文体是自然的当代中文对话：情绪精确但不华丽，该克制的时候克制，该黑暗的时候黑暗但不是为了黑暗本身。

## 好的案例写作应该做到什么

- 每句话都在传递实际的场景信息：情绪、动机、压力、关系状态、潜台词
- 对话听起来像这个角色在这个场景里真的会说的话
- 如果克制是 point 的一部分，那就保持克制
- 风格是用来支撑意义的，不是用来替代意义的

## 应该避免什么

**网文腔污染** (`webnovel_register_contamination`)：像"眸色暗了暗""那是猎人看见猎物的眼神"这类库存短语，语义模糊、情绪泛化。如果出现在没有上下文支撑的地方，应该被当作质量问题而不是风格选择。

**好看的空洞** (`aestheticized_vagueness`)：听起来很沉很文学但实际上没说什么的句子。坏案例不应该能藏在质感后面。

**借来的亲密** (`borrowed_intimacy`)：没有角色逻辑支撑的罐头式诱惑、执念或危险感措辞。

**类型片自动覆盖** (`genre_pastiche_overreach`)：场景不应该因为模型见过这类 pattern 就自动进入"暗黑言情模式"。类型感应该从角色和场景逻辑中来，不是从训练数据里直接搬来。

## 关于黑暗内容

案例可以包含黑暗、占有性、强制性或心理不稳定的材料。但黑暗是内容，不是写作许可证。黑暗场景仍然需要连贯的动机、关系逻辑、可信的声音。

## 底线

如果一句话只是因为闻起来像回收的网文才显得有戏剧感，那它可能不够好。目标不是泛泛的好看，是场景真实。
